# Real-Time E-commerce Funnel Analytics

This repository is a local-first data engineering mono-repo for an interview-ready
e-commerce funnel analytics platform. It focuses on the core data engineering lifecycle:
event generation, Kafka ingestion, Spark streaming and batch processing, Parquet lake
storage, PostgreSQL marts, Airflow orchestration, and data quality checks.

## Architecture

```text
Python Event Generator
        |
        v
Kafka topic: user_events_v1
        |
        v
Spark Structured Streaming
        |
        +--> Bronze Parquet (raw Kafka payloads)
        +--> Silver Parquet (typed, deduplicated events)
        +--> Quarantine Parquet (invalid records)
        |
        v
Spark Batch Aggregations
        |
        +--> Gold Parquet (analytics-ready outputs)
        +--> PostgreSQL staging + analytics schemas
        |
        v
Airflow DAG orchestration + SQL quality checks
```

## Quick Start

1. Install `uv`.
2. Create a local environment from `.env.example`.
3. Sync developer dependencies:

```bash
uv sync --extra dev
```

4. Start local infrastructure:

```bash
docker compose up -d --build
```

5. Generate sample events without Kafka:

```bash
uv run python -m apps.event_generator.main --dry-run --count 10 --seed 7
```

6. Install runtime extras and run jobs locally when needed:

```bash
uv sync --extra dev --extra kafka --extra spark --extra postgres
uv run python -m jobs.streaming.process_events
uv run python -m jobs.batch.build_daily_analytics
uv run python -m jobs.batch.run_quality_checks
```

## Project Defaults

- Kafka topic: `user_events_v1`
- Lake layers: Bronze, Silver, Gold
- Warehouse schemas: `staging`, `analytics`
- Delivery semantics: at-least-once with deduplication on `event_id`
- Invalid events: written to quarantine, never silently dropped

## What Is Implemented Here

- Shared configuration and event contract
- Deterministic and randomized event generation
- Spark streaming job scaffold with Bronze/Silver/Quarantine outputs
- Spark batch job scaffold with analytics mart generation
- PostgreSQL DDL and quality checks
- Airflow DAG that orchestrates the daily batch pipeline
- Unit tests for the shared data contract and core business logic

The repository is intentionally scoped to data engineering only. It excludes dashboards,
ML, MLOps, Terraform, and Kubernetes in v1.
